9. key 를 써야하는 이유
 ['사과', '망고', '포도'] 에서 ['사과', '포도']가 되면 배열의 망고를 삭제했다 라고 생각한다.
 그런데 다르게 생각하면 포도를 삭제하고 두번째 망고를 포도로 바꿨을 수도 있다.
 즉, 어떻게 바뀐건지 결과만 봐서는 알 수 없음.
 [
    {key: 1, '사과'},
    {key: 2, '망고'},
    {key: 3, '포도'},
 ]
 [
    {key: 1, '사과'},
    {key: 3, '포도'},
 ]
 이렇게 있다면 결과만 보고도 어떻게 바뀐것인지 알 수 있음.
 배열의 변화를 리액트에 정확하게 전달하려면 key를 고유한 값으로 전달해야 한다.
