13. 싱글 페이지 애플리케이션(SPA) 이해하기
 a 태그가 있는데 굳이 리액트 라우터를 이용하여 페이지를 이동하는 이유가 뭘까? ==> 우리가 만든게 SPA 이기 때문이다.
 개발자 도구에서 볼 수 있는 root div 안에 있는 것들은 모두 자바스크립트로 리액트가 생성한 HTML 이다.
 이런 방식을 클라이언트 사이드 렌더링이라고 한다.
 클라이언트 - 웹 브라우저
 렌더링 - HTML 을 만드는것
 링크를 클릭할때마다(주소가 바뀔 때마다) 웹 브라우저가 GET 리퀘스트를 보내고 HTML을 받아야 하는데 리퀘스트를 안보낸다.
 Link 컴포넌트에서는 웹 브라우저의 기본 동작 대신에 리액트 라우터에서 경로에 맞는 리액트 컴포넌트를 렌더링하고
 주소창의 글자를 바꾸고 웹 브라우저에 기록을 추가한다. 리퀘스트를 보내지 않고도 자바스크립트를 사용해서 HTML 페이지를 받은 것처럼 흉내내는 것이다.
 ==> SPA ==> 하나의 HTML 파일 안에서 자바스크립트로 여러페이지를 돌아다니는 것처럼 만든 것
 개발모드에서는 알아서 설정이 되어 있지만(리액트 라우터) 클라이언트사이드 렌더링으로 리액트 라우터를 사용하는 경우에는 
 서버에 배포할 때는 따로 설정을 해줘야 한다.
